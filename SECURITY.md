# Security Policy

Last updated: March 10, 2026

## Supported Release Line

The current public distribution repo supports the latest published desktop release line only.

At the moment:

- `v0.1.1` is the current public preview release
- Older preview builds may not receive fixes

## How to report a vulnerability

Please do **not** file sensitive security reports as public GitHub issues.

Use one of these paths instead:

1. GitHub Security Advisories / private vulnerability reporting for this repository, if available
2. A direct private contact through the repository owner profile on GitHub

## What to include

- A clear summary of the issue
- Affected release version
- Reproduction steps
- Impact assessment
- Any screenshots, logs, or receipt excerpts that help explain the issue

## Scope notes

This repository is a public distribution repo, not a source repo. Security reports should focus on:

- The published desktop application
- Release assets
- Installation and update paths
- Local configuration and receipt exposure

## Response expectations

Response times are best effort during the preview stage. Critical issues affecting published release assets will be prioritized first.
